    Sistema de disseny

    Paleta de colors​

    La paleta de Trainium està definida a ui/theme/Color.kt. Segueix una estètica fosca i monocromàtica amb blau elèctric com a color d'accent.

    Tons base (independents del tema)​

    TokenValor HEXÚs principal
    BlueDark#0B1426Fons del tema fosc
    BlueMid#121F3DSuperfície del tema fosc
    BlueDeep#162347Superfície variant del tema fosc
    BlueAccent#2196F3Color primari d'accions (botons, vores actives)
    BlueElectric#448AFFGradient de botons CTA
    BlueSoft#64B5F6Textos secundaris i subtítols en tema fosc
    BlueGlow#332196F3Efectes de brillantor i ombres amb transparència
    BlueLight#BBDEFBFons suaus en tema clar
    WhiteSoft#F0F4FFFons del tema clar

    Tipografia​

    L'escala tipogràfica està definida a ui/theme/Type.kt i utilitza FontFamily.SansSerif (la font sans-serif per defecte del sistema Android, normalment Roboto) per a tots els nivells.

    NivellPesmidaInterletraje
    displayLargeNegre (900)34 sp-0,25 sp
    headlineLargeNegreta (700)28 sp0 sp
    headlineMediumNegreta (700)24 sp0 sp
    titleLargeSemiNegreta (600)20 sp0,15 sp
    titleMediumSemiNegreta (600)16 sp0,15 sp
    bodyLargeNormal (400)16 sp0,5 sp
    bodyMediumNormal (400)14 sp0,25 sp
    labelLargeNegreta (700)14 sp1,2 sp
    labelMediumMitjana (500)12 sp0,5 sp
    labelSmallMitjana (500)11 sp0,5 sp

    El labelLarge amb letterSpacing = 1.2 sp s'usa als botons CTA (com "COMENÇAR ARA", "CONTINUAR") per donar l'efecte de text expandit característic de la interfície.

    Formes​

    El sistema de formes personalitzat PremiumShapes reemplaça l'esquema de cantons per defecte de Material 3:

    NivellRadio d'esquina
    small10 dp
    medium16 dp
    large22 dp
    extraLarge28 dp

    Aquests radis més generosos que els valors per defecte de Material 3 contribueixen a l'aspecte premium arrodonit de targetes i botons.

    Gestió de temes​

    El tema es gestiona en dues capes:

    MaterialTheme (ui/theme/Theme.kt): Defineix el Trainium2Theme composable que aplica el ColorScheme, la Typography i les Shapes segons el paràmetre darkTheme.

    ThemeManager (ThemeManager.kt): Persisteix la preferència de l'usuari a DataStore. La preferència es llegeix com un Flow<Boolean?> i es col·lecciona a MainActivity.kt amb collectAsState(initial = null). El valor inicial null vol dir que la preferència encara no s'ha llegit del disc; en aquest cas, es fa servir el tema del sistema (isSystemInDarkTheme()).

    val savedDarkMode by themeManager.isDarkMode.collectAsState(initial = null)
    val systemTheme = isSystemInDarkTheme()
    var isDarkTheme by remember(savedDarkMode) {
        mutableStateOf(savedDarkMode ?: systemTheme)
    }

    Gradients de fons​

    Totes les pantalles utilitzen gradients verticals en comptes de fons plans. El gradient varia segons el tema actiu:

    Tema fosc:

    Brush.verticalGradient(listOf(BlueDark, BlueMid, BlueDeep))
    // #0B1426 → #121F3D → #162347

    Esborrar el tema:

    Brush.verticalGradient(listOf(Color(0xFFF0F4FF), Color(0xFFE3ECFF), Color(0xFFD6E4FF)))

    Animacions d'entrada​

    Totes les pantalles implementen animacions dentrada escalonades mitjançant LaunchedEffect(Unit) + animateFloatAsState. El patró estàndard és:

    var elementoVisible by remember { mutableStateOf(false) }
    val alpha by animateFloatAsState(
        targetValue = if (elementoVisible) 1f else 0f,
        animationSpec = tween(durationMillis = 500),
        label = "elemento_alpha"
    )

    LaunchedEffect(Unit) {
        delay(100)
        elementoVisible = true
    }

    Els elements es revelen seqüencialment amb retards de 100-300 ms entre cadascun, creant un efecte daparició en cascada.

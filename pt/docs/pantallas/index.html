    Telas

    Esta seção descreve cada elemento que pode ser composto em nível de tela, suas responsabilidades, os parâmetros que recebe e os efeitos colaterais que produz.

    SplashVideoScreen​

    Ficha: SplashVideoScreen.kt Ruta: splash

    Exibe uma tela de boas-vindas com reprodução de vídeo introdutório. O vídeo é carregado de res/raw/video_app.mp4 usando VideoView integrado ao Compose com AndroidView.

    Comportamento:

    • Aparece com um fade-in de 500 ms no início.
    • Após 2 segundos, o indicador de progresso e a marca aparecem na área inferior.
    • Navega automaticamente para loading após a conclusão do vídeo (OnCompletionListener) ou após um tempo limite de 13 segundos se o vídeo não terminar.
    • O fundo é fixo em branco, independentemente do tema do sistema.

    Parâmetros:

    ParâmetroTipoDescrição
    onVideoFinished() -> UnitRetorno de chamada invocado no final do vídeo

    LoadScreen (tela de carregamento)​

    Ruta: loading

    Tela intermediária que verifica se existe uma sessão persistente válida em SecureSessionManager. Não possui elemento de composição próprio, a lógica é executada diretamente no elemento de composição do caminho NavHost loading.

    Comportamento:

    • Leia a sessão salva usando SecureSessionManager.obtenerSesion().
    • Se existir uma sessão válida, navegue diretamente para profile/{nombre}/{isAdmin}/{idUsuario}/{isPremium}.
    • Se não houver sessão, navegue até main.

    Tela Principal​

    Ficha: MainScreen.kt Ruta: main

    Tela de boas-vindas para usuários não autenticados. Mostra o logotipo do Trainium, o slogan e um botão de login. Inclui o seletor de temas no canto superior direito.

    Animações: logotipo com escala + alfa, título com alfa, botão com alfa + deslocamento vertical. Todos os elementos são animados sequencialmente com LaunchedEffect.

    Parâmetros:

    ParâmetroTipoDescrição
    isDarkThemeBooleanControla o tema visual ativo
    onToggleTheme() -> UnitAlternar entre tema claro e escuro
    onNavigateToLogin() -> UnitNavegue até a tela de login

    Tela de Login​

    Ficha: LoginScreen.kt Ruta: login Modelo de visualização: LoginViewModel

    Gerencia a autenticação de usuários existentes usando ID e senha.

    Fluxo de autenticação:

    1. O usuário insere seu ID e senha.
    2. O ViewModel consulta AuthRepository.getUserByDni(dni).
    3. Se um registro for encontrado e a senha corresponder, SecureSessionManager.iniciarSesion(...) será chamado para persistir a sessão.
    4. onLoginSuccess é invocado com o nome, sinalizador admin, id e sinalizador premium do usuário.
    5. Se as credenciais estiverem incorretas, uma mensagem de erro será exibida na tela.

    Parâmetros:

    ParâmetroTipoDescrição
    isDarkThemeBooleanControla o tema visual ativo
    onBack() -> UnitRetornar para MainScreen
    onNavigateToRegister() -> UnitNavegue até o registro
    onNavigateToForgot() -> UnitNavegue até recuperação de senha
    onLoginSuccess(String, Int, Int, Int) -> UnitRetorno de chamada com (nome, isAdmin, userid, isPremium)

    Tela de Registro​

    Ficha: RegisterScreen.kt Ruta: register Modelo de visualização: RegisterViewModel

    Formulário de registro de novo usuário. Colete nome, ID, e-mail, senha e número de telefone e delegue para AuthRepository.insertUser(...).

    Validações implementadas:

    • Todos os campos obrigatórios devem ser preenchidos.
    • O DNI já não pode estar registado (verificação prévia com AuthRepository.userExistsByDni).
    • O novo usuário é inserido com admin=0 e premium=false por padrão.

    Parâmetros:

    ParâmetroTipoDescrição
    isDarkThemeBooleanControla o tema visual ativo
    onBack() -> UnitRetornar para LoginScreen

    Esqueci a senha da tela​

    Ficha: ForgotPasswordScreen.kt Ruta: forgot Modelo de visualização: ForgotPasswordViewModel

    Fluxo de recuperação de senha em duas etapas:

    Etapa 1: O usuário insere seu ID e e-mail. AuthRepository.verifyCredentials(dni, email) é chamado para verificar se o par existe. Se estiver correto, prossiga para a etapa 2.

    Etapa 2: O usuário digita a nova senha e sua confirmação. Se corresponderem, o campo contraseniaHash é atualizado usando AuthRepository.updatePassword(userId, newPass).

    Parâmetros:

    ParâmetroTipoDescrição
    isDarkThemeBooleanControla o tema visual ativo
    onBack() -> UnitRetornar para LoginScreen

    ##Tela de perfil

    Ficha: ProfileScreen.kt Ruta: profile/{nombre}/{isAdmin}/{idUsuario}/{isPremium} Modelo de visualização: ProfileViewModel

    Hub principal do aplicativo após login. Mostra o avatar do usuário, seu nome, seu crachá Premium/Admin e os botões de acesso a cada seção.

    Funcionalidade adicional ao carregar:

    • Recupera dados atualizados do usuário do Supabase para exibir a foto do perfil.
    • Verifique se há alguma reserva cancelada automaticamente devido à manutenção de máquinas para a data atual. Nesse caso, exibe um informativo AlertDialog uma vez por sessão (controlado pela variável global avisoMantenimientoMostradoEnSesion).

    Botões de navegação disponíveis:

    BotãoDisponível paraDestino
    MáquinasTodosmaquinas/{isAdmin}/{idUsuario}
    Minhas ReservasTodosreservas/{isAdmin}/{idUsuario}
    PratosTodosplatos/{isAdmin}/{idUsuario}
    Registro de pesoTodosregistro/{idUsuario}
    Meu perfilTodosedit_profile/{idUsuario}
    SairTodosmain (popUpTo 0) + SecureSessionManager.cerrarSesion()

    Parâmetros:

    ParâmetroTipoDescrição
    nombreStringNome de usuário autenticado
    isAdminBooleanFunção de administrador
    idUsuarioIntID do usuário no banco de dados
    isPremiumBooleanStatus de assinatura premium
    isDarkThemeBooleanControla o tema visual ativo
    onToggleTheme() -> UnitAlternar entre tema claro e escuro
    onLogout() -> UnitRetorno de chamada de logout
    onNavigateToMaquinas(Boolean, Int) -> UnitNavegue até o catálogo de máquinas
    onNavigateToPlatos(Boolean, Int) -> UnitNavegue até o catálogo de pratos
    onNavigateToRegistro(Int) -> UnitNavegue até o registro de peso
    onNavigateToReservas(Boolean, Int) -> UnitNavegue até reservas
    onNavigateToEditProfile(Int) -> UnitNavegue até edição de perfil

    Editar Tela de Perfil​

    Ficha: EditProfileScreen.kt Ruta: edit_profile/{idUsuario} Modelo de visualização: EditProfileViewModel

    Permite ao usuário modificar nome, e-mail, telefone, senha e foto do perfil. A foto é selecionada na galeria do dispositivo usando ActivityResultContracts.GetContent() e convertida para Base64 usando ImageUtils.uriToBase64. Ao salvar, você delega para UsuarioRepository.update(...).

    Parâmetros:

    ParâmetroTipoDescrição
    idUsuarioIntID do usuário para editar
    isDarkThemeBooleanControla o tema visual ativo
    onBack() -> UnitRetornar para ProfileScreen
    onNavigateToHistorial(Int) -> UnitNavegue até o histórico de pagamentos
    onNavigateToPremium() -> UnitNavegue até a seleção do plano Premium

    Catálogo de máquinas de ginástica com funcionalidade de reserva para usuários padrão e gerenciamento de manutenção para administradores.

    A documentação detalhada está localizada em Funcionalidades - Máquinas y reservas.

    Lista de reservas com sistema de filtragem. A documentação detalhada está localizada em Funcionalidades - Máquinas y reservas.

    Tela de Placas​

    Ficha: PlatosScreen.kt Ruta: platos/{isAdmin}/{idUsuario} Modelo de visualização: PlatosViewModel

    Catálogo de pratos nutricionais. A documentação detalhada está localizada em Funcionalidades - Platos y nutrición.

    Tela de Registro​

    Ficha: RegistroScreen.kt Ruta: registro/{idUsuario} Modelo de visualização: RegistroViewModel

    Controle de peso corporal com gráfico de evolução. A documentação detalhada está localizada em Funcionalidades - Registro de peso.

    Tela Historial​

    Ficha: HistorialScreen.kt Ruta: historial/{idUsuario} Modelo de visualização: HistorialViewModel

    Mostra o histórico de pagamentos do usuário classificado por data decrescente. Os pagamentos são debitados de PagoRepository.getByUser(idUsuario).

    Cada item da lista mostra o conceito (tipo de plano), a forma de pagamento, o valor e a data. Em caso de erro de conectividade, será exibida uma mensagem de erro com um botão de nova tentativa. Enquanto os dados estão sendo carregados, uma lista básica é exibida.

    Parâmetros:

    ParâmetroTipoDescrição
    idUsuarioIntID do usuário cujo histórico é consultado
    isDarkThemeBooleanControla o tema visual ativo
    onBack() -> UnitRetornar para EditProfileScreen

# Classes utilitárias e constantes

## AppConfig

**Ficha:** `AppConfig.kt`

Objeto singleton que centraliza todas as constantes de configuração da aplicação. Evite usar literais de string espalhados por todo o código.

```kotlin
object AppConfig {
    // Supabase
    const val SUPABASE_URL = "https://mi-proyecto.supabase.co"
    const val SUPABASE_KEY = "eyJhbGci..."

    // Firebase / DataConnect
    const val FIREBASE_API_KEY = "AIzaSy..."
    const val FIREBASE_APP_ID = "1:123456789:android:abc"
    const val DATACONNECT_URL = "https://europe-southwest1-dataconnect.googleapis.com/..."

    // Sesión persistente
    const val SESSION_PREFS_NAME = "trainium_session"
    const val SESSION_DURATION_MS = 7L * 24L * 60L * 60L * 1000L  // 7 días

    // Imágenes
    const val IMAGE_QUALITY = 70

    // Formatos de fecha
    val FORMAT_ISO_DATE = SimpleDateFormat("yyyy-MM-dd", Locale.getDefault())
    const val FORMAT_DATE_TIME = "yyyy-MM-dd HH:mm"
    const val FORMAT_HOUR_MINUTE = "HH:mm"
}
```

## ImageUtils

**Ficha:** `ImageUtils.kt`

Ele contém duas funções de conversão entre formatos de imagem. É usado para armazenar fotos de perfil, fotos de máquinas e fotos de pratos no Supabase.

###uriToBase64

Converte um URI de imagem selecionado da galeria de dispositivos em uma string Base64 compactada no formato JPEG.

```kotlin
fun uriToBase64(uri: Uri, contentResolver: ContentResolver): String?
```

| Parâmetro | Tipo | Descrição | 
|---|---|---| 
| `uri` | `Uri` | URI da imagem selecionada pelo usuário | 
| `contentResolver` | `ContentResolver` | Resolva do contexto para abrir o fluxo de imagens |

**Valor de retorno:** String Base64 da imagem compactada ou `null` se ocorrer alguma exceção.

**Processo interno:**
1. Abra o `InputStream` do URI usando `contentResolver.openInputStream(uri)`. 
2. Decodifique o fluxo para `Bitmap` com `BitmapFactory.decodeStream`. 
3. Compacte o bitmap para JPEG com qualidade `AppConfig.IMAGE_QUALITY` (70) em um `ByteArrayOutputStream`. 
4. Codifique a matriz de bytes resultante para Base64 com `Base64.encodeToString(byteArray, Base64.DEFAULT)`.

### decodificarBase64ToBitmap

Converte uma string Base64 de volta em um objeto `Bitmap`.

```kotlin
fun decodeBase64ToBitmap(base64Str: String): Bitmap?
```

| Parâmetro | Tipo | Descrição | 
|---|---|---| 
| `base64Str` | `String` | String Base64 representando a imagem |

**Valor de retorno:** `Bitmap` decodificado ou `null` se a string não for Base64 válida ou ocorrer alguma exceção.

## ThemeManager

**Ficha:** `ThemeManager.kt`

Gerencia a persistência de preferências de tema (claro/escuro) usando Jetpack DataStore Preferences.

```kotlin
class ThemeManager(private val context: Context) {
    val isDarkMode: Flow<Boolean?>
    suspend fun setDarkMode(isDark: Boolean)
}
```

`isDarkMode` gera `null` na primeira execução. O coletor em `MainActivity.kt` trata `null` como "usando o tema do sistema".

## NotificationSettingsManager

**Ficha:** `NotificationSettingsManager.kt`

Tanto o avanço de notificação de reserva escolhido pelo usuário quanto o status de ativação global das notificações persistem no DataStore.

```kotlin
class NotificationSettingsManager(private val context: Context) {
    val minutesBefore: Flow<Int>
    val notificationsEnabled: Flow<Boolean>

    suspend fun setMinutesBefore(minutes: Int)
    suspend fun setNotificationsEnabled(enabled: Boolean)

    companion object {
        const val DEFAULT_MINUTES = 30
        val OPTIONS = listOf(10, 15, 30, 60, 120)
    }
}
```

`minutesBefore` gera o valor atual (padrão: 30 minutos). `notificationsEnabled` gera `true` por padrão. A UI de configuração está localizada em `EditProfileScreen`. Consulte [Notificaciones de reservas](/docs/funcionalidades/notificaciones) para uso completo.

## Gerenciador de idiomas

**Ficha:** `data/i18n/LanguageManager.kt`

Persiste o idioma selecionado pelo usuário através do DataStore.

```kotlin
class LanguageManager(private val context: Context) {
    val currentLanguage: Flow<AppLanguage?>
    suspend fun setLanguage(language: AppLanguage)
    fun detectSystemLanguage(): AppLanguage
}
```

`currentLanguage` gera `null` até que o usuário selecione um idioma. `detectSystemLanguage()` mapeia `Locale.getDefault()` para o `AppLanguage` mais próximo, com `SPANISH` como substituto. Consulte [Internacionalización](/docs/arquitectura/i18n) para o sistema completo.

## Administrador do banco de dados

**Ficha:** `DatabaseAdmin.kt`

Singleton que define uma conexão JDBC direta com um banco de dados MySQL/MariaDB local. Esta classe corresponde ao atalho usado durante as fases iniciais de desenvolvimento.

**Esta classe não está ativa na versão de produção**; todas as operações de dados são realizadas por meio de `SupabaseClient`. Está preservado no repositório como referência histórica.

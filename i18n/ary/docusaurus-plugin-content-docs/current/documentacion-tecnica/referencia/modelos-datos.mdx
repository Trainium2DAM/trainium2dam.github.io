#نماذج البيانات

جميع فئات البيانات في الحزمة `models` تمت الإشارة إليها بـ `@Serializable` لـ `kotlinx.serialization`. إنها غير قابلة للتغيير بشكل افتراضي وتستخدم لإلغاء تسلسل استجابات Supabase ولتسلسل نصوص طلبات الإدراج والتحديث.

---

## مستخدم

**فيشيرو:** `models/Usuario.kt` 
**الطبلة سوباباسي:** `usuarios`

```kotlin
@Serializable
data class Usuario(
    @SerialName("id")
    val id: Int = 0,
    val nombre: String,
    val dni: String,
    val contraseniaHash: String,
    val admin: Int,
    val premium: Boolean,
    val email: String? = null,
    val telefono: String? = null,
    val foto: String? = null,
    @SerialName("fecha_ini_prem")
    val fechaInicio: String? = null,
    @SerialName("fecha_fin_prem")
    val fechaFin: String? = null
)
```

| المجال | اكتب | لاغية | الوصف | 
|---|---|---|---| 
| `id` | `Int` | لا | PK التزايدي التلقائي (الافتراضي 0 عند الإدراج) | 
| `nombre` | `String` | لا | الاسم الكامل | 
| `dni` | `String` | لا | يستخدم DNI كبيانات اعتماد لتسجيل الدخول | 
| `contraseniaHash` | `String` | لا | كلمة المرور (نص عادي في هذا الإصدار) | 
| `admin` | `Int` | لا | 0 = المستخدم، 1 = المسؤول | 
| `premium` | `Boolean` | لا | حالة الاشتراك | 
| `email` | `String?` | نعم | البريد الإلكتروني | 
| `telefono` | `String?` | نعم | رقم هاتف الاتصال | 
| `foto` | `String?` | نعم | صورة الملف الشخصي في Base64 | 
| `fechaInicio` | `String?` | نعم | تاريخ بدء الاشتراك المميز (`yyyy-MM-dd`) | 
| `fechaFin` | `String?` | نعم | تاريخ انتهاء الاشتراك المميز (`yyyy-MM-dd`) |

تتم قراءة الحقول `fechaInicio` و `fechaFin` في `ProfileViewModel` لحساب الأيام المتبقية من الاشتراك المميز وعرضها في `ProfileScreen`.

##بيانات الجلسة

**فيشيرو:** `models/SesionData.kt`

```kotlin
data class SesionData(
    val userId: Int,
    val userName: String,
    val isAdmin: Boolean,
    val isPremium: Boolean,
    val token: String = ""
)
```

النموذج المستخدم بواسطة `SecureSessionManager` لتمثيل الجلسة المستمرة للمستخدم. لم تتم إضافة تعليقات توضيحية إليه بـ `@Serializable` لأنه لم يتم تسلسله إلى JSON؛ يتم تخزينه حقلاً بحقل في `SharedPreferences`.

## ريسيرفا

**فيشيرو:** `models/Reserva.kt` 
**الطبلة سوباباسي:** `reservas`

```kotlin
@Serializable
data class Reserva(
    val id: Int? = null,
    @SerialName("id_usuario") val idUsuario: Int,
    @SerialName("id_maquina") val idMaquina: Int,
    val fecha: String,
    @SerialName("hora_inicio") val horaInicio: String,
    @SerialName("hora_fin") val horaFin: String,
    val estado: Boolean = true
)
```

| المجال | اكتب | لاغية | الوصف | 
|---|---|---|---| 
| `id` | `Int?` | نعم | PK (خالية عند الإدراج، تم تعيينها بواسطة Supabase) | 
| `idUsuario` | `Int` | لا | FK → users.id | 
| `idMaquina` | `Int` | لا | FK → Machines.id | 
| `fecha` | `String` | لا | تاريخ الحجز (`yyyy-MM-dd`) | 
| `horaInicio` | `String` | لا | وقت البدء (`HH:mm`) | 
| `horaFin` | `String` | لا | وقت الانتهاء (`HH:mm`). الحد الأقصى 60 دقيقة بعد `horaInicio` | 
| `estado` | `Boolean` | لا | `true` = نشط، `false` = ملغى |

## ريسيرفاكونديتالي

فئة البيانات الموسعة للاستعلامات ذات العلاقات المضمنة (PostgREST الانضمام الضمني).

```kotlin
@Serializable
data class ReservaConDetalles(
    val id: Int,
    val fecha: String,
    @SerialName("hora_inicio") val horaInicio: String,
    @SerialName("hora_fin") val horaFin: String,
    val estado: Boolean,
    @SerialName("usuarios") val usuario: Usuario? = null,
    @SerialName("maquinas") val maquina: Maquina? = null
)
```

يتم الحصول عليه بالعمود `"*, usuarios(*), maquinas(*)"`.

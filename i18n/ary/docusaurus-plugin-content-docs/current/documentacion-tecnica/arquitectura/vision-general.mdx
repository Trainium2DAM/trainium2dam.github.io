# نظرة عامة على الهندسة المعمارية

## النمط المعماري

يتبع Trainium بنية متعددة الطبقات تجمع بين طبقة العرض التقديمي (المواد المركبة + ViewModels)، وطبقة مستودع للوصول إلى البيانات، ومحدد موقع الخدمة كنقطة حقن. معظم منطق الأعمال موجود في ViewModels والمستودعات؛ تقوم طبقة العرض بتفويضهم من خلال المكالمات وتلاحظ حالتهم التفاعلية.

```
Capa de presentación  →  ViewModels  →  Repositorios  →  SupabaseClient / DataConnectClient
```

لا تزال بعض الشاشات الثانوية (السجل، والإصدار المميز) تقوم بتشغيل coroutines مباشرة في `rememberCoroutineScope()`، ولكن التدفق الرئيسي (الأجهزة، والحجوزات، والمصادقة، والملف الشخصي) يعمل بالفعل من خلال نمط ViewModel + Repository.

## الواجهات الخلفية الخارجية

يتفاعل التطبيق مع واجهتين خلفيتين خارجيتين:

**Supabase** بمثابة الواجهة الخلفية الرئيسية. يدير جميع جداول البيانات باستخدام بروتوكول PostgREST. تقوم Supabase Kotlin SDK تلقائيًا بتعيين استجابات JSON لفئات البيانات باستخدام `kotlinx.serialization`. تتم المصادقة مباشرة مقابل الجدول `usuarios` عن طريق التحقق من المعرف وكلمة المرور، وليس من خلال تدفق Supabase Auth (الذي تم تضمين وحدته ولكن لم يتم استخدامها بشكل نشط).

**يتم استخدام Firebase Data Connect** كواجهة خلفية تكميلية عبر عميل HTTP اليدوي (`DataConnectClient.kt`) الذي يستدعي نقطة نهاية GraphQL. تم الإعلان عن Firebase Auth SDK باعتباره تبعية ومتاحًا للترحيل النهائي لنظام المصادقة.

## إدارة الدولة

تتم إدارة حالة واجهة المستخدم بطريقتين اعتمادًا على الشاشة:

**ViewModels (النمط الرئيسي):** تعرض الشاشات ذات التعقيد الأكبر (الأجهزة، والحجوزات، والأطباق، وتسجيل الدخول، والتسجيل، والملف الشخصي) حالتها من خلال متغيرات `mutableStateOf` المعلنة في ViewModel والتي تمت ملاحظتها من الملف القابل للتركيب. ينجو ViewModel من عمليات تدوير الشاشة وتغييرات التكوين.

**الحالة المحلية (الشاشات الثانوية):** تستخدم بعض الشاشات البسيطة (السجل، والشاشات المميزة) `remember` + `mutableStateOf` داخل الجزء القابل للتركيب. في هذه الحالات، لا تنجو الحالة من دوران الشاشة، وهو أمر مقبول نظرًا لأن التطبيق يفرض الاتجاه الرأسي حسب التصميم.

يتم الحفاظ على تفضيل السمة على القرص بواسطة `ThemeManager` (DataStore) وقراءتها في `MainActivity.kt` باعتبارها `Flow<Boolean?>` مجمعة مع `collectAsState`.

يتم الحفاظ على تفضيل اللغة بواسطة `LanguageManager` (DataStore) ويتم حله كـ `Flow<AppLanguage?>` إلى `MainActivity.kt`. إذا لم يقم المستخدم بتحديد لغة، فسيتم اكتشاف لغة النظام باستخدام `languageManager.detectSystemLanguage()`.

## التدويل (i18n)

يستخدم نظام التدويل `CompositionLocal` (`LocalStrings`) لتزويد جميع المواد القابلة للتركيب بإمكانية الوصول إلى السلاسل النصية للغة النشطة، دون الحاجة إلى تمرير السلاسل كمعلمات.

### اللغات المدعومة

| الكود | اللغة | 
|---|---| 
| `es` | الاسبانية | 
| `en` | الإنجليزية | 
| `ca-VALENCIA` | بلنسية | 
| `ca` | الكاتالونية | 
| `fr` | الفرنسية | 
| `ar-MA` | عربية مغربية | 
| `eu` | الباسك | 
| `gl` | الجاليكية | 
| `de` | الألمانية | 
| `pt` | البرتغالية | 
| `ru` | الروسية | 
| `zh` | الصينية |

### تدفق دقة اللغة

```
LanguageManager.currentLanguage (Flow<AppLanguage?>)
    ├── null → languageManager.detectSystemLanguage() (Locale del sistema)
    └── AppLanguage → stringsForLanguage(language) → AppStrings
                                    ↓
                    CompositionLocalProvider(LocalStrings provides currentStrings)
                                    ↓
                        val strings = LocalStrings.current  // en cualquier composable
```

يتم فتح محدد اللغة من أيقونة الكرة الأرضية المتوفرة على `ProfileScreen`، `EditProfileScreen`، `MaquinasScreen` والشاشات الرئيسية الأخرى. يؤدي تحديد لغة في `LanguagePickerDialog` إلى استمرار الكود مع `LanguageManager.setLanguage(language)` ويقوم `MainActivity` القابل للتركيب بإعادة تحميل السلاسل تلقائيًا.

## نظام الإخطار

يقوم التطبيق بجدولة تذكيرات الحجز المحلية باستخدام `AlarmManager`. يتكون النظام من ثلاثة مكونات:

- **`NotificationHelper`:** إنشاء قناة الإشعارات وجدولة التنبيهات الدقيقة (`setExactAndAllowWhileIdle`) وإلغائها. ويحتوي أيضًا على `rescheduleNextReservationNotification`، الذي يستعلم عن الحجوزات المستقبلية للمستخدم ويعيد جدولة التنبيه للحجز التالي. 
- **`NotificationReceiver`:** `BroadcastReceiver` الذي يستقبل التنبيه وينشر الإشعار مع `NotificationCompat`. 
- **`NotificationSettingsManager`:** يستمر الإشعار المسبق الذي اختاره المستخدم (الخيارات: 10 أو 15 أو 30 أو 60 أو 120 دقيقة؛ القيمة الافتراضية: 30 دقيقة) وحالة التنشيط العامة للإشعارات (المنشطة افتراضيًا) في DataStore.

الإذن `POST_NOTIFICATIONS` مطلوب في وقت التشغيل على Android 13+ منذ `MainActivity.onCreate()`. تم الإعلان عن أذونات `SCHEDULE_EXACT_ALARM` و `USE_EXACT_ALARM` في البيان لضمان صحة الإنذارات.

## حقن التبعية

يقوم محدد موقع الخدمة `ServiceLocator` (في `di/ServiceLocator.kt`) بإنشاء مثيل للمستودعات في النموذج `lazy` (مثيل واحد لكل مستودع، يتم إنشاؤه عند الوصول الأول):

```kotlin
object ServiceLocator {
    val authRepository by lazy { AuthRepository() }
    val usuarioRepository by lazy { UsuarioRepository() }
    val maquinaRepository by lazy { MaquinaRepository() }
    val reservaRepository by lazy { ReservaRepository() }
    val platoRepository by lazy { PlatoRepository() }
    val pesoRepository by lazy { PesoRepository() }
    val pagoRepository by lazy { PagoRepository() }
}
```

تصل ViewModels إلى المستودعات من خلال `ServiceLocator`، مع الحفاظ على الاقتران غير المحكم بين الطبقات.

## التكوين المركزي

`AppConfig.kt` يقوم بمركزية جميع ثوابت تكوين التطبيق:

```kotlin
object AppConfig {
    const val SUPABASE_URL = "https://<tu-proyecto>.supabase.co"
    const val SUPABASE_KEY = "<tu-anon-key>"
    const val FIREBASE_API_KEY = "<tu-api-key>"
    const val FIREBASE_APP_ID = "<tu-app-id>"
    const val DATACONNECT_URL = "https://europe-southwest1-dataconnect.googleapis.com/..."
    const val SESSION_PREFS_NAME = "trainium_session"
    const val SESSION_DURATION_MS = 7L * 24L * 60L * 60L * 1000L  // 7 días
    const val IMAGE_QUALITY = 70
    // Formatos de fecha...
}
```

بالقياس، `DbConstants.kt` يعرّف الكائنات `DbTables` و `DbColumns` بأسماء الجداول والأعمدة كثوابت، مع تجنب السلسلة الحرفية المنتشرة في جميع أنحاء الكود.

## ملاحة

تتم إدارة التنقل باستخدام ميزة "إنشاء التنقل". تم الإعلان عن `NavHost` في `MainActivity.kt` ويحدد كافة المسارات. يوجد المخطط التفصيلي في [Grafo de navegación](/docs/arquitectura/grafo-navegacion).

يحل نظام الشكل المخصص `PremiumShapes` محل نظام ركن المادة 3 الافتراضي:

| نيفيل | راديو الإسكينا | 
|---|---| 
| `small` | 10 موانئ دبي | 
| `medium` | 16 موانئ دبي | 
| `large` | 22 موانئ دبي | 
| `extraLarge` | 28 موانئ دبي |

تساهم هذه الأقطار الأكثر سخاءً من القيم الافتراضية للمادة 3 في المظهر المتميز المستدير للبطاقات والأزرار.

## إدارة الموضوع

تتم إدارة الموضوع في طبقتين:

**MaterialTheme (`ui/theme/Theme.kt`):** يحدد `Trainium2Theme` القابل للتركيب الذي يطبق `ColorScheme`، و`Typography`، و`Shapes` وفقًا للمعلمة `darkTheme`.

**ThemeManager (`ThemeManager.kt`):** يحافظ على تفضيلات المستخدم في DataStore. تتم قراءة التفضيل كـ `Flow<Boolean?>` ويتم تجميعه في `MainActivity.kt` مع `collectAsState(initial = null)`. القيمة الأولية `null` تعني أن التفضيلات لم تتم قراءتها من القرص بعد؛ في هذه الحالة، يتم استخدام سمة النظام (`isSystemInDarkTheme()`).

```kotlin
val savedDarkMode by themeManager.isDarkMode.collectAsState(initial = null)
val systemTheme = isSystemInDarkTheme()
var isDarkTheme by remember(savedDarkMode) {
    mutableStateOf(savedDarkMode ?: systemTheme)
}
```

## تدرجات الخلفية

تستخدم كافة الشاشات التدرجات الرأسية بدلاً من الخلفيات المسطحة. يختلف التدرج اعتمادًا على الموضوع النشط:

**موضوع مظلم:**
```kotlin
Brush.verticalGradient(listOf(BlueDark, BlueMid, BlueDeep))
// #0B1426 → #121F3D → #162347
```

**موضوع واضح:**
```kotlin
Brush.verticalGradient(listOf(Color(0xFFF0F4FF), Color(0xFFE3ECFF), Color(0xFFD6E4FF)))
```

## دخول الرسوم المتحركة

تنفذ جميع الشاشات رسومًا متحركة مدخلة متداخلة باستخدام `LaunchedEffect(Unit)` + `animateFloatAsState`. النمط القياسي هو:

```kotlin
var elementoVisible by remember { mutableStateOf(false) }
val alpha by animateFloatAsState(
    targetValue = if (elementoVisible) 1f else 0f,
    animationSpec = tween(durationMillis = 500),
    label = "elemento_alpha"
)

LaunchedEffect(Unit) {
    delay(100)
    elementoVisible = true
}
```

يتم الكشف عن العناصر بشكل تسلسلي مع تأخيرات تتراوح من 100 إلى 300 مللي ثانية بين كل عنصر، مما يؤدي إلى إنشاء تأثير مظهر متتالي.

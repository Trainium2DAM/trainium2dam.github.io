#ترينيوم

Trainium هو تطبيق Android أصلي يهدف إلى الإدارة الشاملة للصالة الرياضية. فهو يسمح للمستخدمين بالتسجيل وحجز الآلات ومراقبة تقدمهم البدني من خلال تسجيل الوزن ومتابعة خطط التغذية. يحتوي على نظام اشتراك متميز يعمل على توسيع الوظائف المتاحة، بالإضافة إلى دور المسؤول مع إمكانية الوصول إلى إدارة كتالوج الآلات والحجوزات النشطة والإشراف على الأطباق الغذائية.

## الميزات الرئيسية

ينفذ التطبيق الوظائف التالية في نسخته النهائية:

- مصادقة المستخدم باستخدام معرف وكلمة المرور، مع استعادة بيانات الاعتماد عن طريق البريد الإلكتروني
- تسجيل مستخدمين جدد مع التحقق من صحة الحقول (الاسم، المعرف، البريد الإلكتروني، الهاتف)
- جلسة مستمرة بين عمليات التمهيد باستخدام `SharedPreferences` مع مدة قابلة للتكوين تبلغ 7 أيام
- حجز الآلات مع تحديد التاريخ والوقت، مع اكتشاف التداخلات والحد الأقصى لمدة ساعة واحدة لكل حجز
- نظام صيانة الآلات مع الإلغاء التلقائي للحجوزات المتأثرة
- إشعارات دفع للتذكير بالحجز المسبق قابلة للتكوين (10 أو 15 أو 30 أو 60 أو 120 دقيقة)
- تصدير التحفظات إلى تقويم الجهاز
- تتبع الوزن مع الرسم البياني لتطور الوقت
- كتالوج الأطباق الغذائية مع نظام الاقتراحات من قبل المستخدم والإشراف من قبل المسؤول
- الاشتراك المميز بثلاثة أنواع أسعار (شهرية ونصف سنوية وسنوية) وتسجيل الدفع مع مؤشر الأيام المتبقية
- تحرير الملف الشخصي باستخدام صورة المستخدم المخزنة في Base64
- دعم السمات الفاتحة والداكنة مع ثبات التفضيل عبر DataStore
- **دعم متعدد اللغات** مع الكشف التلقائي عن لغة النظام والاختيار اليدوي (تتوفر 12 لغة)
- شاشة البداية متحركة مع تشغيل فيديو تمهيدي
- التحميل بشاشات الهيكل العظمي أثناء استعلامات API

## مكدس التكنولوجيا

| طبقة | تكنولوجيا | 
|---|---| 
| اللغة | كوتلين | 
| واجهة المستخدم | Jetpack Compose + المادة 3 | 
| الملاحة | الملاحة يؤلف 2.7.7 | 
| الواجهة الخلفية الرئيسية | Supabase (PostgreSQL) عبر SDK الرسمي | 
| الخلفية البديلة | اتصال بيانات Firebase (GraphQL) | 
| المصادقة | مصادقة Firebase (معلنة) | 
| الثبات المحلي | تفضيلات Jetpack DataStore + SharedPreferences | 
| التدويل | نظام السلسلة الخاص في Compose (i18n) | 
| الإخطارات | مدير التنبيه + NotificationCompat | 
| بناء النظام | Gradle (Kotlin DSL) | 
| بيئة تطوير متكاملة | أندرويد ستوديو |

## هيكل التوثيق

**التثبيت والإعداد** يغطي متطلبات بيئة التطوير، واستنساخ المستودع، وإعداد بيانات اعتماد Supabase وFirebase اللازمة لبناء المشروع.

**الهندسة المعمارية** تصف بنية الوحدة والرسم البياني للتنقل وطبقة المستودع وقرارات التصميم المعتمدة.

**قاعدة البيانات** توثق مخطط جدول Supabase وعلاقات الكيانات ومخطط Firebase Data Connect GraphQL.

**الشاشات** تصف كل شاشة من شاشات التطبيق ومسؤولياتها وتدفقات التنقل.

**الوظائف** تفاصيل تنفيذ كل كتلة وظيفية رئيسية.

**واجهة برمجة التطبيقات والخدمات** تشرح التكامل مع Supabase وFirebase Auth وFirebase Data Connect.

**التصميم** يوثق نظام التصميم ولوحة الألوان والطباعة ومكونات واجهة المستخدم القابلة لإعادة الاستخدام.

**النشر** يشير إلى خطوات إنشاء تصحيح الأخطاء أو إصدار APK.

**المرجع** يجمع جرد نماذج البيانات والثوابت وفئات المرافق.

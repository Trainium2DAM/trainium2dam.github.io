## Gaien kudeaketa

Gaia bi geruzatan kudeatzen da:

**MaterialTheme (`ui/theme/Theme.kt`):** `ColorScheme`, `Typography` eta `Shapes` `darkTheme` parametroaren arabera aplikatzen dituen `Trainium2Theme` konposagarria definitzen du.

**ThemeManager (`ThemeManager.kt`):** erabiltzailearen hobespenak mantentzen ditu DataStore-n. Hobespena `Flow<Boolean?>` gisa irakurtzen da eta `MainActivity.kt`-n biltzen da `collectAsState(initial = null)`-rekin. Hasierako balioak `null` esan nahi du hobespena ez dela oraindik diskotik irakurri; kasu horretan, sistemaren gaia (`isSystemInDarkTheme()`) erabiltzen da.

```kotlin
val savedDarkMode by themeManager.isDarkMode.collectAsState(initial = null)
val systemTheme = isSystemInDarkTheme()
var isDarkTheme by remember(savedDarkMode) {
    mutableStateOf(savedDarkMode ?: systemTheme)
}
```

## Atzeko planoko gradienteak

Pantaila guztiek gradiente bertikalak erabiltzen dituzte atzealde lauen ordez. Gradientea gai aktiboaren arabera aldatzen da:

**Gai iluna:**
```kotlin
Brush.verticalGradient(listOf(BlueDark, BlueMid, BlueDeep))
// #0B1426 → #121F3D → #162347
```

**Garbitu gaia:**
```kotlin
Brush.verticalGradient(listOf(Color(0xFFF0F4FF), Color(0xFFE3ECFF), Color(0xFFD6E4FF)))
```

## Sarrera animazioak

Pantaila guztiek sarrerako animazio mailakatuak ezartzen dituzte `LaunchedEffect(Unit)` + `animateFloatAsState` erabiliz. Eredu estandarra hau da:

```kotlin
var elementoVisible by remember { mutableStateOf(false) }
val alpha by animateFloatAsState(
    targetValue = if (elementoVisible) 1f else 0f,
    animationSpec = tween(durationMillis = 500),
    label = "elemento_alpha"
)

LaunchedEffect(Unit) {
    delay(100)
    elementoVisible = true
}
```

Elementuak sekuentzialki erakusten dira bakoitzaren artean 100-300 ms-ko atzerapenekin, kaskadako itxura efektua sortuz.

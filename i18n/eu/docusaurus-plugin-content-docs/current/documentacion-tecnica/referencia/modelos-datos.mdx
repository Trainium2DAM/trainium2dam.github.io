# Datu ereduak

`models` paketeko datu-klase guztiak `@Serializable` edo `kotlinx.serialization`-rekin adierazita daude. Lehenespenez aldaezinak dira eta Supabase erantzunak deserializatzeko eta txertatzeko eta eguneratzeko eskaeren gorputzak serializatzeko erabiltzen dira.

---

## Erabiltzailea

**Fichero:** `models/Usuario.kt` 
**Tabla Supabase:** `usuarios`

```kotlin
@Serializable
data class Usuario(
    @SerialName("id")
    val id: Int = 0,
    val nombre: String,
    val dni: String,
    val contraseniaHash: String,
    val admin: Int,
    val premium: Boolean,
    val email: String? = null,
    val telefono: String? = null,
    val foto: String? = null,
    @SerialName("fecha_ini_prem")
    val fechaInicio: String? = null,
    @SerialName("fecha_fin_prem")
    val fechaFin: String? = null
)
```

| Eremua | Mota | Deusezgarria | Deskribapena | 
|---|---|---|---| 
| `id` | `Int` | Ez | PK autoincremental (lehenetsia 0 txertatzean) | 
| `nombre` | `String` | Ez | Izen osoa | 
| `dni` | `String` | Ez | Saio-hasteko kredentzial gisa erabilitako NAN | 
| `contraseniaHash` | `String` | Ez | Pasahitza (testu arrunta bertsio honetan) | 
| `admin` | `Int` | Ez | 0 = erabiltzailea, 1 = administratzailea | 
| `premium` | `Boolean` | Ez | Harpidetzaren egoera | 
| `email` | `String?` | Bai | Posta elektronikoa | 
| `telefono` | `String?` | Bai | Harremanetarako telefono zenbakia | 
| `foto` | `String?` | Bai | Profileko irudia Base64 | 
| `fechaInicio` | `String?` | Bai | Premium harpidetzaren hasiera-data (`yyyy-MM-dd`) | 
| `fechaFin` | `String?` | Bai | Premium harpidetzaren amaiera-data (`yyyy-MM-dd`) |

`fechaInicio` eta `fechaFin` eremuak `ProfileViewModel`-n irakurtzen dira Premium harpidetzaren gainerako egunak kalkulatzeko eta `ProfileScreen`-n bistaratzeko.

## Erreserba

**Fichero:** `models/Reserva.kt` 
**Tabla Supabase:** `reservas`

```kotlin
@Serializable
data class Reserva(
    val id: Int? = null,
    @SerialName("id_usuario") val idUsuario: Int,
    @SerialName("id_maquina") val idMaquina: Int,
    val fecha: String,
    @SerialName("hora_inicio") val horaInicio: String,
    @SerialName("hora_fin") val horaFin: String,
    val estado: Boolean = true
)
```

| Eremua | Mota | Deusezgarria | Deskribapena | 
|---|---|---|---| 
| `id` | `Int?` | Bai | PK (txertatzeko nulua, Supabasek esleitutakoa) | 
| `idUsuario` | `Int` | Ez | FK → erabiltzaileak.id | 
| `idMaquina` | `Int` | Ez | FK → makinak.id | 
| `fecha` | `String` | Ez | Erreserba data (`yyyy-MM-dd`) | 
| `horaInicio` | `String` | Ez | Hasiera ordua (`HH:mm`) | 
| `horaFin` | `String` | Ez | Amaiera-ordua (`HH:mm`). Gehienez 60 minutu `horaInicio` | 
| `estado` | `Boolean` | Ez | `true` = aktibo, `false` = bertan behera |

## ErreserbaConDetalles

Kapsulatutako erlazioak dituzten kontsultetarako datu-klase hedatua (PostgREST JOIN inplizitua).

```kotlin
@Serializable
data class ReservaConDetalles(
    val id: Int,
    val fecha: String,
    @SerialName("hora_inicio") val horaInicio: String,
    @SerialName("hora_fin") val horaFin: String,
    val estado: Boolean,
    @SerialName("usuarios") val usuario: Usuario? = null,
    @SerialName("maquinas") val maquina: Maquina? = null
)
```

`"*, usuarios(*), maquinas(*)"` zutabearekin lortzen da.

# ♿ Criterios de Accesibilidad y Usabilidad Aplicados

Se han incorporado diversos criterios de diseño para asegurar que la aplicación sea intuitiva (Usabilidad) y operable por la mayor cantidad de usuarios (Accesibilidad).

## 1. Usabilidad (UX)

| Criterio UX | Aplicación en el Diseño | Impacto en el Usuario |
| :--- | :--- | :--- |
| **Feedback Inmediato** | Uso de barras y gráficos de progreso (ej. Reto de peso , Gráfico de peso). | El usuario comprende rápidamente su estado y evolución, lo que fomenta la motivación. |
| **Consistencia de Navegación** | La **Barra de Navegación Inferior** es un elemento constante y predecible. | Reduce la curva de aprendizaje; el usuario sabe dónde encontrar las funciones clave en todo momento. |
| **Escaneo Rápido** | Agrupamiento de información en *Cards* y uso de encabezados claros (Control de peso, Dieta y nutrición, etc.). | Permite al usuario escanear la pantalla y encontrar la información sin leerla por completo. |
| **Flexibilidad y Eficiencia**| Inclusión de una **Reserva Rápida**, **atajo a estadisticas** en el Dashboard. | Los usuarios experimentados pueden acceder a la función más usada con menos clics. |

## 2. Accesibilidad (A11y)

| Criterio A11y | Aplicación en el Diseño | Impacto en el Usuario |
| :--- | :--- | :--- |
| **Contraste de Color** | Uso de color Púrpura Oscuro (`#71006c`) como color primario de acción sobre fondos blancos o grises claros. | Mejora la legibilidad para usuarios con baja visión, cumpliendo (o apuntando a) las directrices WCAG. |
| **Targets Táctiles** | Los elementos clickeables (Botones, *Cards* de selección) son grandes y espaciados. | Facilita la interacción táctil precisa, crucial para usuarios con temblores o en dispositivos pequeños. |
| **Campos de Formulario Claros** | Etiquetas y *placeholders* explicativos (`@ Usuario`, `Peso (kg)`, `Introduce el DNI`). | Ayuda a usuarios de *screen readers* y reduce errores de entrada de datos. |
| **Información de Referencia** | El **IMC** y **% de grasa** se presentan con una leyenda visualmente clara (Bajo, Normal, Sobrepeso, Obeso). | Los usuarios pueden interpretar rápidamente sus métricas de salud con contexto. |

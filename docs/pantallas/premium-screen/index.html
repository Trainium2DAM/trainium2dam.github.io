PremiumSelectionScreen

Archivo: app/src/main/java/com/example/trainium2/PremiumSelectionScreen.kt

Descripción​

Pantalla de contratación de la suscripción Premium. El usuario elige un plan, selecciona un método de pago e introduce los datos de pago. Al confirmar, se actualiza el estado premium del usuario y se registra el pago.

Firma del Composable​

@Composable
fun PremiumSelectionScreen(
    idUsuario: Int,
    onBack: () -> Unit,
    onSuccess: () -> Unit
)

Planes disponibles​

PlanPrecioDuración
Mensual9,99 €1 mes
Semestral49,99 €6 meses
Anual89,99 €12 meses

Los planes se definen como una lista de Triple<String, Double, Int> (nombre, precio, meses).

Métodos de pago soportados​

  • Tarjeta de crédito / débito: formulario con número de tarjeta, fecha de vencimiento (MM/YY) y CVV.
  • PayPal: sin formulario adicional (flujo simplificado).

Validaciones del formulario de tarjeta​

CampoValidación
Número de tarjeta16 dígitos
Fecha de vencimientoFormato MM/YY
CVV3 dígitos

Operaciones de base de datos​

Registrar el pago​

INSERT INTO pagos (id_usuario, monto, fecha_pago, tipo, metodo_pago)
VALUES (?, ?, ?, ?, ?)

La fecha_pago se genera con SimpleDateFormat("yyyy-MM-dd") del día actual.

Activar estado Premium​

UPDATE usuario
SET PREMIUM = 1, FECHA_INI_PREM = ?, FECHA_FIN_PREM = ?
WHERE ID = ?
  • FECHA_INI_PREM: fecha actual.
  • FECHA_FIN_PREM: fecha actual + meses del plan seleccionado (calculado con Calendar.add(Calendar.MONTH, meses)).

Flujo de compra​

Usuario selecciona plan
    │
    └─ Usuario selecciona método de pago
        │
        ├─ Tarjeta → rellena formulario
        └─ PayPal  → sin formulario adicional
            │
            └─ Pulsa "Confirmar pago"
                │
                ├─ Validaciones locales
                └─ scope.launch(Dispatchers.IO)
                    ├─ INSERT INTO pagos
                    ├─ UPDATE usuario SET PREMIUM = 1, fechas
                    └─ withContext(Main)
                        ├─ Toast "✅ ¡Ahora eres Premium!"
                        └─ onSuccess() → navega de vuelta

Animaciones​

  • Header, tarjetas de planes y formulario de pago: fade + scale de entrada con tween(600).
  • Cada plan selected tiene un borde BlueAccent y escala ligeramente al estar seleccionado.
